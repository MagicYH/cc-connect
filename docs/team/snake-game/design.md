# 终端贪吃蛇游戏 — 技术设计

## 目标

用 Python + curses 实现一个终端贪吃蛇游戏，单文件约 200 行，无外部依赖。

## 方案

- **运行时**: Python 3.8+, curses 标准库
- **单文件**: `snake.py`，直接 `python snake.py` 运行
- **游戏循环**: 非阻塞 `getch()` + `timeout()` 控制帧率

## 关键接口 / 数据流

### 数据结构

```python
snake: list[tuple[int, int]]   # 蛇身坐标，头部在前
food: tuple[int, int]           # 食物坐标
direction: tuple[int, int]      # 当前移动方向 (dy, dx)
score: int                      # 当前分数
paused: bool                    # 暂停状态
game_over: bool                 # 游戏结束状态
```

### 游戏循环

```
1. 读取按键 → 更新 direction（禁止 180° 掉头）
2. 计算新头部位置 = 当前头部 + direction
3. 碰撞检测：
   - 撞墙 → game_over
   - 撞自身 → game_over
4. 吃食物检测：
   - 新头部 == food → 分数+1，生成新食物，不删尾
   - 否则 → 删除尾部
5. 插入新头部到 snake
6. 渲染画面
7. 等待下一帧
```

### 按键映射

| 按键 | 动作 |
|------|------|
| ↑ / w / W | 上 |
| ↓ / s / S | 下 |
| ← / a / A | 左 |
| → / d / D | 右 |
| Space / p / P | 暂停/继续 |
| r / R | 重新开始（game over 后） |
| q / Q | 退出 |

### 渲染

- 边框: `curses.box()` 或手动画 `#` 边框
- 蛇身: `O`，蛇头: `@`
- 食物: `*`
- 状态栏: 底部显示分数、暂停提示、操作说明

## 边界条件与异常处理

1. **180° 掉头保护**: 方向键禁止反向（如正在向右时不能直接向左）
2. **食物不与蛇身重叠**: 生成食物时循环随机直到找到空位
3. **终端窗口过小**: 初始化时检查 `stdscr.getmaxyx()`，不足 20×40 则提示退出
4. **curses 资源清理**: 使用 `wrapper()` 确保异常时终端恢复正常
5. **帧率**: `timeout(100)` ~10 FPS，游戏结束后关闭 timeout 阻塞等待输入

## 风险与代价

- curses 在 Windows 上需 `windows-curses` 包，非标准库 — 在代码头部注释说明
- 单线程模型下按键缓冲可能导致多帧方向跳变 — 通过只取最新方向缓解

## 验收标准

1. `python snake.py` 启动后显示游戏画面，蛇可移动
2. 方向键和 WASD 均可控制方向
3. 吃到食物后蛇身增长、分数+1
4. 撞墙或撞自身后显示 Game Over，分数可见
5. Game Over 后按 r 可重新开始
6. Space/P 可暂停和恢复
7. q/Q 可退出游戏
8. 退出后终端恢复正常（无乱码）
9. 单文件，无第三方依赖（除 Windows 需 windows-curses）
