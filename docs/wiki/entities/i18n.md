# I18n

The internationalization system in `core/i18n.go` that manages all user-facing strings.

## How It Works

1. Define a `MsgKey` constant: `MsgStarting MsgKey = "starting"`
2. Add translations for all 5 languages (EN, ZH, ZH-TW, JA, ES)
3. Use `e.i18n.T(MsgKey)` or `e.i18n.Tf(MsgKey, args...)` in engine code

The `I18n` struct is concurrency-safe with `sync.RWMutex`.

## Project Relation

All user-facing strings (status messages, card titles, error descriptions, slash command responses) must go through i18n. No hardcoded English strings in engine or platform code.
